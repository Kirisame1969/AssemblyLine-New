# 🏭 AssemblyLine 项目脚本目录 (Project Script Directory)
## 1. Data 层 - 纯数据与配置 (Model)
这一层负责存储静态配置（ScriptableObjects）和运行时纯 C# 数据，不包含任何逻辑运算或表现层引用。

### 📄 静态配置 (ScriptableObjects)
- ItemDefinition.cs: 物品定义图纸。配置物品名称、图标、基础价格等静态属性。
- RecipeDefinition.cs: 加工配方图纸。定义输入原料、产出物品、加工耗时及电力/热量等（预留）。
- MachineShellProfile.cs: 机箱外壳图纸。定义机箱的逻辑宽高、内部死区（DeadCells）和预设隔断墙。
- ModuleDefinition.cs: 模块定义图纸。配置模块形状、占用格子及提供的增益效果（Buff）。
- ModuleCatalog.cs: 模块目录表。用于 UI 侧边栏显示的模块集合索引。

### 📄 运行时纯 C# 数据 (POCO Classes)
- ItemData.cs: 物品实例数据。记录物品在传送带上的进度、当前所在格子等状态。
- BeltData.cs: 传送带单元数据。记录方向及所属的连通条带（Strip）。
- StripData.cs: 连通条带逻辑包。管理一组相连传送带的整体移动速度和颜色。
- GridCell.cs: 网格单元容器。地图上每个格子的底层载体，持有对物品、传送带、机箱区域的引用。
- PlayerEconomyData.cs: 玩家经济账户数据。记录资金总量等持久化数值。

### 📄 机器系统子命名空间 (AssemblyLine.Data.Machine)
- MachinePrimitives.cs: 机器基础基元。定义加工队列容器、旋转枚举及物理隔断墙结构体。
- MachineModuleData.cs: 模块数据抽象基类。处理异形模块的空间占用计算与内部连通性数学判定。
- MachineShellData.cs: 机箱实例数据。汇总内部模块、路由索引及动态属性（RecalculateStats），无 GameObject 引用。
- MachineCoreModules.cs: 核心逻辑模块。包含多流水线处理核心、采购终端、交付仓库等特定数据类。
- MachineStandardModules.cs: 标准组件模块。包含输入/输出匣及普通占位增益模块。

---

## 2. Core 层 - 核心仲裁与模拟 (Controller)
这一层是游戏的“大脑”，负责处理 Tick 步进逻辑、物理规则仲裁和数据读写。
- SimulationController.cs: 全局模拟总控。作为主时钟驱动 Tick 步进，控制游戏倍速，管理活跃物品列表。
- MachineManager.cs: 机器核心管理器。负责模块放置/拆卸的规则仲裁，以及 Tick 时驱动所有机器的生产与 I/O 交换。
- GridManager.cs: 网格系统管理器。负责世界坐标与逻辑网格的转换，维护地图数据的完整性。
- StripManager.cs: 物流拓扑管理器。实时维护传送带的连通性变化，自动合并/拆分逻辑条带（Strip）。
- EconomyManager.cs: 经济系统管理器。负责资金的增减逻辑，触发金钱变动后的数据广播。

---

## 3. UI 与表现层 - 渲染与交互 (View)
这一层负责监听数据变化、实例化预制体及捕获玩家输入。
- InteractionController.cs: 大世界交互总控。处理鼠标点击/拖拽、传送带建造、物品抓取，并管理大世界中机箱底板和端口的视觉映射。
- CameraController.cs: 大世界摄像机控制。实现基于鼠标中键的拖拽平移、滚轮缩放及边界限制。
- MachineGUIController.cs: 机器组装面板 View。将 MachineShellData 渲染为 2D 交互界面，处理内部模块的排布交互。
- UICell.cs: UI 网格单元。处理机箱内部界面的悬停高亮、颜色染色。
- UIModuleEntry.cs: 模块列表项。侧边栏中单个模块的拾取入口。
- UITabButton.cs: UI 标签按钮。负责面板内页面切换的视觉反馈。
- UIZoomPanController.cs: 内部界面缩放控制器。负责机箱内部组装网格的缩放与漫游逻辑。
- EconomyUIController.cs: 经济界面 View。同步显示玩家资金数值。
- FloatingTextVisual.cs: 飘字视觉逻辑。处理加钱/扣钱时的动态跳字及淡出效果。
- UIManager.cs: 全局 UI 状态调度。管理不同面板的开启/关闭优先级。
- LanguangManager.cs: 视觉辅助管理器。可能用于处理选中效果或特定的透明遮罩视觉。

---

## ⚠️ 架构备忘
- 数据隔离：Assets/Scripts/Data/ 目录下的内容应保持绝对干净，不应包含 using UnityEngine;（除非使用 Vector2Int 等数学结构）。
- 职责分配：若以后新增“电力系统”，需在 Data 层加 PowerData，Core 层加 PowerManager，View 层加 PowerUIController。
